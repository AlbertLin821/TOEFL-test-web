# 02. 軟體需求規格書（SRS：Software Requirements Specification）

> 文件版本：v1.0  
> 系統名稱：TOEFL-style 四技能英語模擬測驗平台  
> 架構模式：B2B Multi-tenant SaaS  
> 考卷模式：固定整卷式考卷

---

## 1. 系統目的

本系統提供補習班、學校及教育機構建立與執行四技能英語模擬考試，支援學生線上完成 Reading、Listening、Writing、Speaking，並在交卷後自動批改客觀題、使用 AI 批改寫作與口說，最後產生四科整合成績報告並寄送 Email。

---

## 2. 系統範圍

### 2.1 系統包含

- 使用者登入與角色權限
- 多機構資料隔離
- 班級與學生管理
- 固定整卷式考卷管理
- 考試排程與指派
- 學生作答介面
- 硬體檢查流程
- Reading / Listening 自動批改
- Writing / Speaking AI 批改
- 成績報告產生
- Email 通知
- 老師端成績管理
- 管理端使用量與方案管理

### 2.2 系統不包含

- ETS 官方成績認證
- 官方 TOEFL 報名或送分功能
- 第一版不支援大型題庫隨機抽題
- 第一版不支援完整線上監考 AI proctoring
- 第一版不支援個人線上刷卡購買

---

## 3. 使用者角色

| 角色 | 說明 |
|---|---|
| Platform Admin | 平台最高管理者，可管理所有機構、方案、AI prompt 與系統設定 |
| Organization Admin | 機構管理者，可管理自己機構的老師、學生、班級與考試 |
| Teacher | 老師，可管理班級、指派考試、查看成績、補充評語 |
| Student | 學生，可參加考試、查看報告 |

---

## 4. 功能需求

## FR-001 使用者登入

### 描述
使用者可使用 Email + Password 登入系統。

### 驗收標準

- 正確帳密可登入。
- 錯誤帳密不可登入。
- 停用帳號不可登入。
- 登入後依角色導向不同首頁。

---

## FR-002 多機構資料隔離

### 描述
系統必須支援多個 organization，每個 organization 的資料不可互相存取。

### 驗收標準

- A 機構老師不可查看 B 機構學生。
- A 機構學生不可查看 B 機構考卷或報告。
- API 層必須檢查 organization_id。

---

## FR-003 班級管理

### 描述
機構管理者或老師可建立班級並加入學生。

### 驗收標準

- 可建立班級。
- 可編輯班級名稱。
- 可加入與移除學生。
- 可查看班級學生列表。

---

## FR-004 學生帳號管理

### 描述
機構管理者可建立學生帳號，支援單筆建立與批次匯入。

### 驗收標準

- 可單筆新增學生。
- 可用 CSV / Excel 匯入學生。
- Email 不可重複。
- 匯入錯誤資料需回傳錯誤清單。

---

## FR-005 固定整卷式考卷管理

### 描述
每份考卷包含 Reading、Listening、Writing、Speaking 四科完整內容。

### 驗收標準

- 考卷可建立版本。
- 考卷版本發布後不可直接覆蓋。
- 學生作答時綁定特定 exam_version_id。
- 舊版本成績不可因新版本修改而變動。

---

## FR-006 考試排程

### 描述
老師可將考卷指派給班級或學生，並設定開放時間、截止時間與補考規則。

### 驗收標準

- 學生只會看到自己被指派的考試。
- 未到開放時間不可開始。
- 超過截止時間不可開始。
- 可設定 max_attempts。

---

## FR-007 硬體檢查

### 描述
學生開始正式考試前必須完成耳機音量與麥克風錄音測試。

### 驗收標準

- 可播放測試音檔。
- 可請求麥克風權限。
- 可錄製測試音訊。
- 錄音成功後才可進入正式考試。
- 使用者拒絕麥克風權限時需顯示處理方式。

---

## FR-008 Reading 作答

### 描述
學生可完成 Reading section，包括閱讀文章、選擇題、填空題等。

### 驗收標準

- 進入題目後倒數開始。
- 可 Back / Next。
- 可 Review。
- 可 Hide Time / Show Time。
- 同一 module 內可返回題目。
- 進入下一 module 後不可返回上一 module。

---

## FR-009 Listening 作答

### 描述
學生可完成 Listening section，包括聽音選擇題、對話題、公告題與學術演講題。

### 驗收標準

- 音檔播放期間選項灰階不可點選。
- 音檔播放完後選項啟用。
- Listening 只有 Next，沒有 Back。
- 可調整音量。
- 可設定音檔只播放一次。

---

## FR-010 Writing 作答

### 描述
學生可完成 Writing section，包括 Build a Sentence、Write an Email、Academic Discussion。

### 驗收標準

- Build a Sentence 支援拖曳排序。
- Email 題支援文字輸入。
- Academic Discussion 支援文字輸入。
- 顯示字數統計。
- 支援 Hide Word Count。
- 支援 Cut / Paste / Undo / Redo。
- 顯示倒數時間。

---

## FR-011 Speaking 作答

### 描述
學生可完成 Speaking section，包括 Listen and Repeat 與 Virtual Interview。

### 驗收標準

- 系統可播放題目音檔。
- 系統可自動開始錄音。
- 時間到自動停止錄音。
- 顯示 Stop Speaking。
- 錄音檔成功上傳。
- 每題錄音檔與 item_id 正確對應。

---

## FR-012 自動儲存

### 描述
學生作答期間，系統應自動儲存答案與目前進度。

### 驗收標準

- 文字答案定期儲存。
- 選擇題點選後儲存。
- section 狀態需記錄。
- 斷線重登可恢復到最後有效狀態。

---

## FR-013 交卷

### 描述
學生完成所有 section 後可交卷，或時間到由系統自動交卷。

### 驗收標準

- 交卷後不可再修改答案。
- 系統建立 grading_job。
- attempt 狀態改為 submitted 或 grading。
- 客觀題立即批改。

---

## FR-014 Reading / Listening 自動批改

### 描述
系統根據 answer_key 自動計算客觀題分數。

### 驗收標準

- 單選題可判斷正確與錯誤。
- 填空題可依設定判斷大小寫或同義答案。
- Build a Sentence 可依正確排序判斷。
- 批改結果可追溯到每題。

---

## FR-015 Writing / Speaking AI 批改

### 描述
系統將 Writing / Speaking 送入 AI worker 批改，取得 JSON 格式分數與評語。

### 驗收標準

- AI 批改不可阻塞學生前端。
- AI 回傳結果需符合 schema。
- AI 批改失敗需 retry。
- 超過 retry 上限需標示人工處理。
- 需記錄 model、prompt_version、token_usage。

---

## FR-016 成績報告

### 描述
系統產生整合四科的單一成績報告。

### 驗收標準

- 顯示學生資料。
- 顯示考試資料。
- 顯示四科分數。
- 顯示客觀題統計。
- 顯示 AI 評語。
- 可由老師補充人工評語。
- 可匯出 PDF。

---

## FR-017 Email 通知

### 描述
報告完成後系統寄送 Email 給學生，並依設定寄送給老師或機構管理者。

### 驗收標準

- 報告完成後自動寄送。
- Email 中包含成績摘要與報告連結。
- 寄送成功或失敗需記錄。
- 寄送失敗可重試。

---

## FR-018 老師端成績管理

### 描述
老師可查看班級學生歷次考試成績與個別報告。

### 驗收標準

- 可依班級篩選。
- 可依考試篩選。
- 可查看單一學生報告。
- 可新增老師評語。
- 可匯出 Excel。

---

## 5. 非功能需求

## NFR-001 效能

- 系統需支援多名學生同時作答。
- 作答儲存 API 不可造成明顯延遲。
- AI 批改需使用 queue 背景處理。
- 大檔案上傳需避免阻塞 API 主流程。

## NFR-002 安全性

- 密碼需使用 Argon2 或 bcrypt 雜湊。
- Session cookie 使用 HttpOnly、Secure、SameSite。
- OpenAI API key 只能存在後端。
- 音檔、報告 PDF 使用 signed URL。
- 所有資源存取需檢查 organization_id。

## NFR-003 可用性

- 支援最新版本 Chrome、Edge、Safari 桌機版。
- Speaking 錄音功能需在 HTTPS 正式環境運作。
- 錯誤訊息需以使用者可理解方式呈現。

## NFR-004 可維護性

- 後端模組化。
- API 文件化。
- Prompt 版本化。
- DB migration 版本化。
- 重要事件記錄 audit log。

## NFR-005 可擴充性

- 未來可支援題庫抽題。
- 未來可支援個人購買。
- 未來可支援多語言 UI。
- 未來可支援不同 AI 模型。

## 6. 使用情境 Use Cases

## UC-001 機構管理者建立班級

1. Organization Admin 登入。
2. 進入班級管理。
3. 建立班級。
4. 匯入學生名單。
5. 系統建立學生帳號並加入班級。

### 例外
- Email 重複：該筆匯入失敗並列出錯誤。
- 欄位缺漏：匯入失敗並提示欄位格式。

---

## UC-002 老師指派考試

1. Teacher 登入。
2. 選擇班級。
3. 選擇考卷版本。
4. 設定開放時間、截止時間、補考次數。
5. 發布考試。

### 例外
- 考卷未發布：不可指派。
- 開放時間晚於截止時間：不可儲存。

---

## UC-003 學生完成完整模考

1. Student 登入。
2. 選擇可作答考試。
3. 完成硬體檢查。
4. 完成 Reading。
5. 完成 Listening。
6. 完成 Writing。
7. 完成 Speaking。
8. 交卷。
9. 等待批改。
10. 收到報告通知。

### 例外
- 中途斷線：重新登入後恢復。
- 錄音失敗：提示重新測試或聯絡老師。
- 時間到：自動交卷。

---

## UC-004 AI 批改失敗

1. 學生交卷後建立 grading_job。
2. AI worker 呼叫 OpenAI API。
3. 若 API 超時或回傳格式錯誤，系統 retry。
4. 超過上限後標記為 manual_review_required。
5. 通知老師處理。

---

## 7. 錯誤處理

| 情境 | 系統處理 |
|---|---|
| 網路斷線 | 保留最後儲存進度，允許重登恢復 |
| 時間到 | 自動提交當前已儲存答案 |
| AI 批改失敗 | Retry，超過上限轉人工處理 |
| Email 寄送失敗 | 記錄失敗並 retry |
| 錄音權限被拒 | 顯示瀏覽器權限設定教學 |
| 學生重複開啟考試 | 以 active attempt 為準，避免重複作答 |
