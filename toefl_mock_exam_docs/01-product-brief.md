# 01. 產品願景與商業需求說明（BRD / Product Brief）

> 文件版本：v1.0  
> 產品類型：TOEFL-style 四技能英語模擬測驗平台  
> 目標使用者：補習班、學校、語言中心、未來團體帳號客戶  
> 技術方向：React + Node.js/Express + REST API + PostgreSQL + OpenAI API

---

## 1. 產品定位

本產品是一套 **TOEFL-style 四技能英語模擬考試平台**，提供補習班、學校、語言中心與教育機構建立、指派、管理與批改完整英語模擬測驗。

平台支援：

- Reading 閱讀
- Listening 聽力
- Writing 寫作
- Speaking 口說

第一版產品採用 **固定整卷式考卷模式**：

> 每一份考卷已經完整組裝好 Reading、Listening、Writing、Speaking 四個 section。  
> 系統不在第一版實作大型題庫抽題或隨機組卷。

---

## 2. 產品要解決的問題

目前補習班或學校在做托福類型模考時，常見問題包括：

1. **四技能考試流程分散**
   - 閱讀、聽力可以用線上選擇題處理。
   - 口說錄音、寫作批改通常還要另外收檔、人工批改。

2. **老師批改負擔高**
   - 口說與寫作人工評分需要時間。
   - 學生無法即時拿到完整四技能報告。

3. **班級與成績管理麻煩**
   - 常用 Excel、Google 表單、雲端硬碟、Email 分散管理。
   - 老師不容易追蹤學生歷次模考表現。

4. **商業化困難**
   - 若要賣給多間補習班或學校，需要支援多機構帳號、方案額度、學生席位與 AI 批改成本控管。

---

## 3. 產品願景

打造一個可以讓教育機構快速部署四技能英語模考的 SaaS 平台，使補習班或學校能夠：

- 快速安排完整模考
- 自動批改閱讀與聽力
- 使用 AI 批改口說與寫作
- 自動產生四科整合報告
- 自動寄送 Email 通知
- 管理學生、班級、老師與歷史成績
- 未來可透過團體帳號授權商業化

---

## 4. 目標客群

### 4.1 第一階段

| 客群 | 使用目的 |
|---|---|
| 補習班 | 給學生進行托福類型模考與課後檢測 |
| 語言中心 | 建立四技能線上測驗流程 |
| 學校英文課程 | 作為校內英語能力測驗或課程前後測 |
| 老師個人工作室 | 小班制學生模考管理 |

### 4.2 第二階段

| 客群 | 使用目的 |
|---|---|
| 大型教育機構 | 多班級、多老師、多學生管理 |
| 學校行政單位 | 團體帳號與成績報表 |
| 個人考生 | 自行購買單次模考或訂閱方案 |

---

## 5. 商業模式

### 5.1 建議商業模式

第一版建議採用：

> **B2B 團體帳號授權 + 額度制**

也就是先賣給補習班、學校、語言中心，而不是一開始就做個人線上刷卡購買。

---

## 6. 計價方式

建議將方案拆成三種額度：

| 額度類型 | 說明 |
|---|---|
| 學生席位數 | 一個機構可建立多少學生帳號 |
| 考試次數 | 學生可完成幾次完整四技能模考 |
| AI 批改次數 | Writing / Speaking 批改會消耗 AI 額度 |

### 6.1 範例方案

| 方案 | 對象 | 內容 |
|---|---|---|
| Starter | 小型補習班 | 50 學生席位、100 次考試、100 次 AI 批改 |
| Pro | 中型補習班 | 200 學生席位、500 次考試、500 次 AI 批改 |
| School | 學校或大型機構 | 客製席位、客製考試次數、客製 AI 額度 |
| Add-on | 加購 | 額外 AI 批改次數、額外考試次數 |

---

## 7. 高階功能需求

### 7.1 管理端

平台或機構管理者可以：

- 管理機構帳號
- 管理老師帳號
- 管理學生帳號
- 管理班級
- 管理考卷
- 管理方案與額度
- 查看 AI 使用量與成本
- 查看考試統計

### 7.2 老師端

老師可以：

- 建立與管理班級
- 指派考卷給班級或學生
- 設定考試開放時間與截止時間
- 查看學生作答狀態
- 查看成績報告
- 補充老師人工評語
- 匯出 Excel 或 PDF 報告

### 7.3 學生端

學生可以：

- 登入平台
- 查看被指派的考試
- 完成硬體檢查
- 完成 Reading、Listening、Writing、Speaking
- 查看歷史成績
- 下載 PDF 報告
- 收到 Email 成績通知

## 8. 第一版產品範圍

### 8.1 MVP 必做

| 模組 | 功能 |
|---|---|
| 帳號系統 | Email + Password 登入、角色權限 |
| 多機構架構 | Organization / Tenant 隔離 |
| 班級管理 | 老師建立班級與學生名單 |
| 固定整卷考卷 | 每份考卷包含四科 |
| 考試排程 | 指派考卷給班級 |
| 學生考試端 | 硬體檢查、四科作答、倒數計時 |
| 自動批改 | Reading / Listening / Build a Sentence |
| AI 批改 | Writing / Speaking |
| 成績報告 | 四科整合報告 |
| Email 通知 | 批改完成後寄信 |
| 老師端報表 | 查詢成績、補充評語、匯出 |

### 8.2 MVP 暫不做

| 暫緩功能 | 原因 |
|---|---|
| 大型題庫抽題 | 第一版已確認使用固定整卷 |
| 個人線上購買 | 金流與稅務流程可後補 |
| 高強度線上監考 | 初期先做基本防作弊紀錄 |
| 複雜 AI 自動出題 | 先穩定考試與批改流程 |
| 手機版考試 | 四技能模考建議先鎖定桌機與筆電 |

---

## 9. 產品風險

| 風險 | 說明 | 對策 |
|---|---|---|
| 商標風險 | TOEFL 為 ETS 商標 | 對外使用 TOEFL-style、模擬測驗、非官方聲明 |
| AI 評分穩定性 | AI 分數可能有波動 | 使用固定 prompt、JSON schema、版本控管與老師覆核 |
| AI 成本 | 口說與寫作會產生 API 成本 | 設定 AI 批改額度與重批限制 |
| 考試中斷 | 學生斷線可能影響考試 | 自動儲存、後端時間控制、可恢復 |
| 錄音失敗 | 瀏覽器權限或設備問題 | 考前硬體檢查與明確錯誤提示 |

---

## 10. 成功指標

第一版產品成功的判斷標準：

- 學生可以完成一份完整四技能考卷。
- 系統可以正確自動批改 Reading / Listening。
- 系統可以成功呼叫 AI 批改 Writing / Speaking。
- 系統可以產生一份完整四科報告。
- 系統可以寄出 Email 通知。
- 老師可以查看班級學生的成績。
- 不同機構資料不可互相存取。
- 考試中斷後可以恢復到合理狀態。
